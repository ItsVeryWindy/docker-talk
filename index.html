Understanding Docker

A High Level Overview

Created by Oliver Neal

Before Shipping Containers. Inefficient. Moved by hand. Fragile Broke. Took Long.
After Shipping Containers. Packed in safely. Moved by crane. Hours instead of days.

Docker allows for a portable, lightweight environments that can be run in isolation, to allow for a high degree of repeatablility and consistency.

This makes it applications easy to deploy and environments easier to share.

Like a VM:

  • Own process space
  • Own file system
  • Own network interface
  • Install stuff on it
  • Run applications inside it

Not like a VM:

  • Uses host kernel
  • Can't run different OS than host (Windows=Windows,Linux=Linux)
  • Shared resources

Images

Similar to classes/templates

Contain what's required to run

  • OS
  • Application
  • Dependencies

Images can be based off other images

Containers

Instance of an image

The thing that runs the application

Registry

Similar to nuget/npm/ruby gems etc.

But for images

Host

Contains the docker daemon - Does all the heavy lifting

  • Talk to it via docker client
  • Pulls down images
  • Builds images
  • Runs Containers

Contains the images

Contains the containers

Benefits

Easier to manage versioning

Closer to production

Easy to get application up an running
